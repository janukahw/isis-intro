Meta / the making of

Built by talking to a machine.

Every page, diagram and animation on this site was written by an AI coding agent — Anthropic’s Claude Code — steered in plain English by one human. No frameworks, no build step, no designer, no copywriter. This page shows the receipts: the toolchain, the loop, and the actual prompts.

H.1 — What was in the room

The toolchain

agent

Claude Code

An AI agent that lives in the terminal: it reads the whole repo, edits files, runs commands, drives a browser, and commits — while the human reviews diffs and steers in plain English.

model

Claude Fable 5

The Anthropic model behind the work — every commit in the repo, from the very first to this page, is co-signed, so the claim is checkable: git log → Co-Authored-By: Claude Fable 5.

constitution

CLAUDE.md

A rules file the agent re-reads every session: vanilla HTML/CSS/JS only, zero dependencies, must work from file://, accessibility is a gate, every physics fact traced to a source. The agent polices itself against it.

plugins · skills

frontend-design + modern-web-guidance

Two installed Claude Code plugins: Anthropic’s frontend-design (pushes a bold, distinctive look) and the Chrome team’s modern-web-guidance (current platform best practice). When they pulled in different directions, the conflict was audited and the resolution written into CLAUDE.md.

mcp server

Playwright

The agent’s hands on a real browser, via the Model Context Protocol: load each page, read the console, screenshot diagrams, seek animations to exact frames, emulate reduced-motion and five screen widths.

deploy

GitHub Pages

Push to main and the site is live in about a minute. The agent watches the deploy via the GitHub API, then re-checks the live URL with a cold cache before calling it done.

H.2 — How every change shipped

The loop

Every feature on this site went around the same loop. A plain-English prompt becomes a concrete plan; the agent builds it; then — before anything ships — it verifies its own work in a real browser: zero console errors, screenshots inspected frame by frame, the reduction playground run end-to-end, reduced-motion checked for a fully readable static page, no horizontal overflow from 375 px to 1920 px, and everything tested over both file:// and a local server.

Only then does it ship: commit, push, watch the GitHub Pages build, and spot-check the live site. Failures go back around the loop, not into production.

H.3 — The actual conversation

Real prompts, real diffs

The site was built across a morning’s worth of sessions. The first session — the one that produced all six pages from nothing — had its transcript auto-cleaned minutes after the site shipped, so its prompts can’t be quoted; what survives is its output, commit c90882e at 08:20: “Initial release: interactive ISIS intro site … No build step; works from file:// and any static server. Facts checked against isis.stfc.ac.uk, the ISIS Practical Guide, and Mantid docs.”

The later sessions are on record. These are quoted verbatim — typos included, because they never mattered. Each one-liner below a prompt is what landed on main.

can you check if front-end-design skill and google-web-design skill conflict with each other

a plugin audit: overlap and tension points mapped, verdict written into CLAUDE.md as a design-direction section

i want to explore top 5 places where motion or effects will add value … do an adversial review for each one you find — reasons not to do — then only priortize

six candidates killed with written reasons; the four survivors built: traveling waves, panel cues, plot tweens

im not keen on page scroll based animations — happy to switch to a different more straightforward method

the whole story page rebuilt from scroll-scrubbing to time-based loops; an entire browser-fallback subsystem deleted

the bar chart underneath it, can that be animated aswell … you will need to show significantly more neutrons to align with the counts of each bar

30 generated neutrons per pulse; the histogram builds one visible step per hit, left to right in arrival order

Scene 02 — the machine — i see two red circles

a one-line fix, verified in the browser, live a few minutes later

The middle prompt is the interesting one: a design U-turn mid-project. The agent didn’t patch around it — it rewrote the motion system, deleted the dead code, and updated its own rules file so the old approach can’t creep back.

H.4 — What kept it honest

Guardrails

A written constitution. CLAUDE.md sets hard rules the agent checks its own work against — architecture (“no dependencies, no build step”), an accessibility bar (keyboard paths, labels on every diagram, reduced-motion must leave a fully readable static page — true of every animation on this site, including the ones here), and a verification checklist run before every push.

Facts trace to sources. Physics claims trace to isis.stfc.ac.uk or the Mantid docs, and schematic models carry a “simplified” footnote. That bar applied even to decoration: the time-of-flight race stamps are real numbers from λ×v = 3956, and the detector-histogram animation only shipped once each bar grew by exactly one step per neutron hit.

And a human in the loop. Every diff was reviewed; every push was an explicit human decision. The agent proposes, verifies and argues — the human decides.
